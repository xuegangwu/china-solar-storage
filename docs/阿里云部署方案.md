## 📅 部署步骤

### Step 1: 购买和配置阿里云资源（30 分钟）

#### 1.1 创建 ECS 实例

```bash
# 登录阿里云控制台
https://ecs.console.aliyun.com/

# 创建实例
1. 选择地域：华东 1（杭州）或 华北 2（北京）
2. 镜像：Ubuntu 22.04 64 位
3. 实例规格：ecs.c6.large（2 核 4G）
4. 存储：100GB ESSD 云盘
5. 带宽：5Mbps（按使用流量）
6. 安全组：
   - 放行 80（HTTP）
   - 放行 443（HTTPS）
   - 放行 22（SSH）
   - 放行 3001（API 服务）
```

#### 1.2 创建 RDS PostgreSQL

```bash
# 登录 RDS 控制台
https://rds.console.aliyun.com/

# 创建实例
1. 选择地域：与 ECS 相同
2. 数据库版本：PostgreSQL 14
3. 实例规格：2 核 4G
4. 存储：100GB
5. 网络类型：VPC（与 ECS 同 VPC）
6. 白名单：添加 ECS 内网 IP
```

#### 1.3 创建 OSS Bucket

```bash
# 登录 OSS 控制台
https://oss.console.aliyun.com/

# 创建 Bucket
1. Bucket 名称：solar-storage-map（全局唯一）
2. 地域：与 ECS 相同
3. 读写权限：私有
4. 开启 CDN 加速（可选）
```

#### 1.4 域名备案

```bash
# 如果域名未备案
1. 登录阿里云备案系统
2. 提交备案申请
3. 等待审核（7-20 个工作日）
4. 备案通过后解析域名
```

### Step 2: 配置服务器环境（30 分钟）

#### 2.1 SSH 连接服务器

```bash
# 连接服务器
ssh root@<ECS 公网 IP>

# 或配置 SSH 密钥
ssh-copy-id root@<ECS 公网 IP>
```

#### 2.2 安装基础软件

```bash
#!/bin/bash
# deploy-env.sh

# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs

# 安装 Git
apt install -y git

# 安装 Nginx
apt install -y nginx

# 安装 PM2（进程管理）
npm install -g pm2

# 安装 PostgreSQL 客户端
apt install -y postgresql-client

# 安装 Redis 客户端（可选）
apt install -y redis-tools

# 验证安装
node -v
npm -v
nginx -v
pm2 -v
```

#### 2.3 配置防火墙

```bash
# 配置 UFW 防火墙
ufw allow 22/tcp      # SSH
ufw allow 80/tcp      # HTTP
ufw allow 443/tcp     # HTTPS
ufw allow 3001/tcp    # API 服务
ufw enable

# 查看状态
ufw status
```

### Step 3: 部署应用代码（20 分钟）

#### 3.1 克隆代码

```bash

mkdir -p /var/www/solar-storage-map
cd /var/www/solar-storage-map

# 克隆代码
git clone https://github.com/xuegangwu/china-solar-storage.git .

# 或上传代码（使用 scp）
# scp -r ./china-solar-storage/* root@<ECS_IP>:/var/www/solar-storage-map/
```

#### 3.2 安装后端依赖

```bash
cd /var/www/solar-storage-map/server

# 安装依赖
npm install

# 创建环境配置文件
cp .env.example .env

# 编辑配置文件
nano .env
```

#### 3.3 配置环境变量

```bash
# .env 文件内容
NODE_ENV=production
PORT=3001

# 数据库配置（从阿里云 RDS 获取）
DB_HOST=<RDS 内网地址>

DB_NAME=solar_storage
DB_USER=postgres
DB_PASSWORD=<你的数据库密码>

# JWT 密钥（生成随机字符串）
JWT_SECRET=<openssl rand -base64 32>

# OSS 配置（从阿里云 OSS 获取）
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_BUCKET=solar-storage-map
OSS_ACCESS_KEY_ID=<你的 AccessKey ID>
OSS_ACCESS_KEY_SECRET=<你的 AccessKey Secret>

# 邮件配置（可选）
SMTP_HOST=smtp.qq.com
SMTP_PORT=587
SMTP_USER=<你的邮箱>
SMTP_PASS=<你的邮箱授权码>
```

### Step 4: 配置数据库（15 分钟）

#### 4.1 连接 RDS

```bash
# 从 ECS 连接 RDS
psql -h <RDS 内网地址> -U postgres -d postgres

# 输入密码
```

#### 4.2 创建数据库和用户

```sql
-- 创建数据库
CREATE DATABASE solar_storage;

-- 创建用户
CREATE USER solar_user WITH PASSWORD '<强密码>';

-- 授权
GRANT ALL PRIVILEGES ON DATABASE solar_storage TO solar_user;

-- 切换数据库
\c solar_storage

-- 创建 schema
CREATE SCHEMA IF NOT EXISTS public;

-- 授权 schema
GRANT ALL ON SCHEMA public TO solar_user;
```

#### 4.3 运行数据库迁移

```bash
cd /var/www/solar-storage-map/server

# 运行迁移脚本（需要创建）
node database/migrate.js
```

### Step 5: 配置 Nginx（15 分钟）

#### 5.1 创建 Nginx 配置文件

```bash
nano /etc/nginx/sites-available/solar-storage-map
```

```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    
    # 前端静态文件
    location / {
        root /var/www/solar-storage-map/web;
        index index.html;
        try_files $uri $uri/ /index.html;
        
        # Gzip 压缩
        gzip on;
        gzip_types text/plain text/css application/json application/javascript;
        gzip_min_length 1000;
    }
    
    # API 反向代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
    
    # 静态资源（OSS 或本地）
    location /assets {
        alias /var/www/solar-storage-map/web/assets;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}

# HTTPS 配置（申请 SSL 证书后）
# server {
#     listen 443 ssl http2;
#     server_name your-domain.com;
#     
#     ssl_certificate /etc/nginx/ssl/your-domain.crt;
#     ssl_certificate_key /etc/nginx/ssl/your-domain.key;
#     
#     # ... 其他配置同上
# }
```

#### 5.2 启用配置

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/solar-storage-map /etc/nginx/sites-enabled/

# 删除默认配置
rm /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
systemctl enable nginx
```

### Step 7: 配置 SSL 证书（15 分钟）

#### 7.1 申请免费 SSL 证书

```bash
# 方法 1：阿里云申请免费证书
# 登录 SSL 证书控制台
# https://yundun.console.aliyun.com/?p=cas

# 方法 2：使用 Let's Encrypt
apt install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.com -d www.your-domain.com
```

#### 7.2 配置 HTTPS

```nginx
# 在 Nginx 配置中添加
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # SSL 优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;
    ssl_ciphers HIGH:!aNULL:!MD5;
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

```bash
#!/bin/bash
# backup.sh

# 备份数据库
pg_dump -h <RDS 地址> -U solar_user solar_storage > /backups/db-$(date +%Y%m%d).sql

# 备份文件
tar -czf /backups/files-$(date +%Y%m%d).tar.gz /var/www/solar-storage-map

# 上传到 OSS
ossutil cp -r /backups oss://solar-storage-map/backups/

# 清理 30 天前的备份
find /backups -name "*.sql" -mtime +30 -delete
find /backups -name "*.tar.gz" -mtime +30 -delete

echo "✅ 备份完成！"
```

## 💰 成本优化建议

### 1. 使用抢占式实例
- 节省 50-70% 成本
- 适合开发测试环境

### 2. 购买预留实例券
- 1 年期节省 30-40%
- 3 年期节省 50%+

### 3. 使用 CDN 加速
- 减少源站流量
- 提升访问速度

### 4. 开启自动缩放
- 根据负载自动调整
- 降低闲置成本

### 5. 使用 OSS 归档存储
- 冷数据转归档
- 节省 70% 存储成本
