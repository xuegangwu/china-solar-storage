## 🐛 问题描述

用户反馈：越南光储电站投资地图页面显示不正常

---

## 🔍 问题诊断

### 问题 1：后端 API 错误

**错误日志**:
```
获取省份数据错误：Error: no such column: total
    at /home/admin/openclaw/workspace/projects/china-solar-storage/backend/routes/provinces.js:14:26
```

**原因**: `routes/provinces.js` 第 14 行 SQL 查询使用了不存在的字段 `total`

**问题代码**:
```javascript
const provinces = db.exec('SELECT * FROM provinces ORDER BY total DESC')[0];
```

**数据库结构**:
- provinces 表没有 `total` 字段
- `total` 实际存储在 `scores` JSON 字段中：`scores.total`

### 问题 2：前端字段引用错误

**问题代码** (`vietnam-map.html`):
```javascript
// 第 194 行
const sorted = [...provincesData].sort((a, b) => b.total - a.total);

// 第 201 行
<div class="province-score">总分：${province.total.toFixed(1)}</div>
```

**实际情况**:
- API 返回的数据结构：`{ scores: { total: 61 }, ... }`
- 前端错误引用：`province.total`
- 正确引用：`province.scores.total`

---

## ✅ 修复方案

### 修复 1：后端 SQL 查询

**文件**: `backend/routes/provinces.js`

**修改**:
```diff
- const provinces = db.exec('SELECT * FROM provinces ORDER BY total DESC')[0];
+ const provinces = db.exec('SELECT * FROM provinces ORDER BY grade DESC, name ASC')[0];
```

**说明**: 改为按 `grade` 和 `name` 排序，避免使用不存在的 `total` 字段

### 修复 2：前端字段引用

**文件**: `web/vietnam-map.html`

**修改**:
```bash
sed -i 's/province\.total/province.scores.total/g' vietnam-map.html
```

**影响位置**:
- 第 194 行：排序逻辑
- 第 201 行：分数显示

## ⚠️ 注意事项

### 数据结构一致性

API 返回的省份数据结构：
```json
{
  "id": 1,
  "name": "宁顺省",
  "scores": {
    "total": 80,
    "resource": 25,
    "policy": 20,
    ...
  },
  "grade": "A",
  "country": "VN",
  ...
}
```

**重要**:
- `total` 在 `scores` 对象内，不在根级别
- 前端引用必须使用 `province.scores.total`

### 其他可能受影响的页面

检查所有使用 `province.total` 的文件：
```bash
grep -r "province\.total" /home/admin/openclaw/workspace/projects/china-solar-storage/web/
```

如有其他文件使用，需要同样修复。
